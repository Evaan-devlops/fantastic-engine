SOP: Retrieve OneTrust Production Script for an Existing Website

Goal

Retrieve the "data-domain-script" value for an existing OneTrust website.

Goal identifier:

"retrieve_data_domain_script"

Input

"target_website_url={{input.target_website_url}}"

Output

"data_domain_script={{output.data_domain_script}}"

Scope

- The target website already exists in OneTrust.
- Adding a new website is not supported by this SOP.
- Username, password, MFA, OTP and other authentication details are entered manually by the user.
- Credentials must never be stored in the SOP, task files, logs or generated artifacts.

Capability 1: authenticate_onetrust

This is the entry capability.

1. Open:
   
   "https://uat-de.onetrust.com/cookies/websites"

2. Wait until the OneTrust login page is fully rendered and the login controls are visible.

3. Click the "Next" button.

4. If a separate authentication page or browser tab opens, pause execution for "MANUAL_AUTH".

5. Ask the user to complete username, password, MFA and any other authentication steps manually in the browser.

6. Do not read, copy, record or store authentication values.

7. After authentication succeeds, return to the OneTrust browser tab when necessary.

8. Resume only when both conditions are true:
   
   - The current URL equals:
     
     "https://uat-de.onetrust.com/cookies/websites"
   
   - The OneTrust Websites page is completely rendered and the Search textbox is visible.

9. If the post-authentication page cannot be verified, stop safely with:
   
   "AUTHENTICATION_NOT_VERIFIED"

Capability 2: locate_existing_website

1. Locate the Search textbox on the Websites page.

2. Fill the Search textbox with:
   
   "{{input.target_website_url}}"

3. Press Enter to run the search.

4. Wait until the results table is visible and completely rendered.

5. Locate the result row whose Website URL exactly equals:
   
   "{{input.target_website_url}}"

6. Do not select a partial or similar URL.

7. If no exact matching row is found, stop safely with:
   
   "WEBSITE_NOT_FOUND"

8. Read the Scan status value from the exact matching row.

9. If Scan status is "Pending", wait and check it again.

10. Wait for a maximum of 300 seconds, checking every 5 seconds.

11. Continue only when Scan status exactly equals:
    
    "Completed"

12. If the status does not become "Completed" within 300 seconds, stop safely with:
    
    "SCAN_NOT_COMPLETED"

13. Click the Website URL link in the exact matching row.

Capability 3: retrieve_production_script

1. Wait until the website-detail page is completely rendered.

2. Locate the More options button represented by the three-dot ellipsis icon.

3. Click the More options button.

4. Wait until the dropdown menu is visible.

5. Select:
   
   "Copy production scripts"

6. Wait until the Production scripts modal is visible and completely rendered.

7. Locate the production script containing the "data-domain-script" attribute.

8. Extract only the value of the "data-domain-script" attribute.

9. Store the extracted value as:
   
   "{{output.data_domain_script}}"

10. Report the extracted value as the successful task output.

11. Finish with terminal outcome:
    
    "COMPLETED"
